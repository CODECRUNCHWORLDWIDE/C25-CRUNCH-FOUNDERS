# Mini-Project — Build and Run Your MVP

> Build the smallest experiment that tests your riskiest assumption, run it with real people, and record the result and your decision. This MVP (or a more polished prototype built on it) is one of your capstone deliverables.

**Estimated time:** ~1.75 hours of write-up (plus the build in homework).

## What you will produce

A folder `mvp/` in your repo containing:

1. **The artifact** — the page, form, prototype, or a description of the concierge/Wizard-of-Oz process, plus a link or screenshots.
2. **`mvp/EXPERIMENT.md`** — the experiment record:
   - The riskiest assumption, as a hypothesis.
   - MVP type and why it was the cheapest test.
   - The single metric and the pre-committed threshold.
   - How and with whom you ran it (no private identifiers).
   - The result vs. the threshold.
   - Validated learning and a persevere/pivot decision.

---

## Acceptance criteria

- [ ] `mvp/` contains the artifact (or a clear description) with a link/screenshot.
- [ ] `mvp/EXPERIMENT.md` states the hypothesis, MVP type, metric + pre-set threshold, result, learning, and decision.
- [ ] The MVP actually tests the stated riskiest assumption (not a side feature).
- [ ] The build was cheap and fast (days, not weeks).
- [ ] Committed.

## Rubric

| Criterion | Weight |
|-----------|------:|
| The MVP tests the stated riskiest assumption | 30% |
| Threshold was set before launch | 20% |
| Result is reported honestly against the threshold | 20% |
| MVP was genuinely minimal / cheap | 15% |
| Decision (persevere/pivot) is justified by the number | 15% |

# Exercise 1 — Riskiest Assumption

**Goal:** Surface the assumptions your idea depends on, rank them, and choose the one your MVP will test — plus the cheapest MVP type to test it.

**Estimated time:** 45 minutes.

---

## What to produce

Create `exercises/exercise-01-riskiest-assumption.md`:

```markdown
# Riskiest assumption — <your idea>

## Assumption list
| Assumption | Uncertainty (1–5) | Fatal if false? (1–5) | Risk score |
|------------|:-----------------:|:---------------------:|:----------:|
| People will adopt this | 5 | 5 | 25 |
| ... | | | |

## My riskiest assumption
(The highest risk-score row, stated as a testable hypothesis.)

## Chosen MVP type
(Fake door / concierge / Wizard of Oz / clickable prototype / single-feature build — and why it is the cheapest test of THIS assumption.)

## Success metric and threshold (set NOW)
Metric: ___  ·  Success = at least ___
```

---

## Acceptance criteria

- [ ] File present; at least five assumptions scored on both axes.
- [ ] One riskiest assumption chosen, stated as a hypothesis.
- [ ] An MVP type chosen with justification.
- [ ] A single metric and a pre-committed threshold.
- [ ] Committed.

---

## Hints

- If your chosen MVP requires weeks of coding, you probably picked a build when a fake door or concierge would do.
- The threshold must be set before launch. No moving it afterward.

# Exercise 1 — Pitch Outline

**Goal:** Draft your 3-minute pitch slide by slide, with a timing budget and the deliverable that backs each claim.

**Estimated time:** 50 minutes.

---

## What to produce

Create `exercises/exercise-01-pitch-outline.md`:

```markdown
# Pitch outline — <company>

| Beat | Seconds | What I say (one line) | Backed by (deliverable) |
|------|--------:|-----------------------|-------------------------|
| Problem | 20 | | — |
| Evidence | 40 | | discovery dossier |
| Solution | 30 | | MVP / prototype |
| Model | 30 | | canvas + unit economics |
| Traction | 30 | | GTM one-pager |
| Ask | 20 | | — |
Total: 170s (leaves buffer under 3:00)

## My three hard-question answers
1. "How do you know this is real?" →
2. "Why you / why now?" →
3. "Can this make money / how big?" →

## My riskiest assumption (stated calmly)
```

---

## Acceptance criteria

- [ ] File present; all six beats with a one-line script and a timing budget summing to ≤180s.
- [ ] Each evidence-bearing beat names the deliverable that backs it.
- [ ] Three hard-question answers drafted.
- [ ] Riskiest assumption stated.
- [ ] Committed.

---

## Hints

- If a beat has no deliverable behind it, either get the evidence or cut the claim.
- Leave a few seconds of buffer; live pitches always run slightly long.
